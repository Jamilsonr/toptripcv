# Fluxos

## Férias
1. Colaborador cria pedido (intervalo de datas).
2. Sistema calcula dias úteis (com feriados configurados).
3. Valida saldo disponível e conflitos básicos (opcional no MVP).
4. Submete pedido.
5. Gestor aprova/rejeita.
6. RH aprova/rejeita.
7. Se aprovado: deduz saldo e publica no calendário.
8. Notificação para o colaborador em cada transição.

## Licenças/Ausências
1. Colaborador cria pedido (tipo, datas/horas, motivo/anexo futuro).
2. Submete pedido.
3. Gestor aprova/rejeita.
4. RH aprova/rejeita.
5. Notificação em cada transição.

## Comunicação interna
- RH publica comunicado/notícia/evento.
- Colaboradores visualizam no dashboard e recebem notificação in-app.
